# 작업 목록(task.md) 생성 워크플로우

기획 단계(`PLANNING`)에서 `implementation_plan.md`의 **구현 순서(✅ 구현 순서)** 섹션이 확정되면, 아래 절차에 따라 `task.md` 파일을 생성합니다.

## 실행 조건

- `implementation_plan.md`가 존재하고, **구현 순서** 섹션이 작성되어 있어야 합니다.
- 사용자가 기획안을 승인했거나, 구현 시작을 요청한 시점에 실행합니다.

## 생성 절차

1. `implementation_plan.md`의 **구현 순서** 섹션을 읽습니다.
2. 각 구현 단계를 `## 단계번호. 단계명` 형식의 섹션으로 변환합니다.
3. 각 단계 아래에 세부 작업 항목을 `- [ ] 세부 작업 내용` 체크박스 형식으로 나열합니다.
   - 파일 생성 작업은 파일 경로를 포함합니다. (예: `- [ ] \`lib/auth.ts\` — 테스트 유저 헬퍼 함수`)
   - 설치/설정 작업은 구체적인 패키지명이나 도구명을 포함합니다.
4. 파일 상단에 프로젝트 제목을 `# 프로젝트명 - 작업 목록` 형식으로 작성합니다.
5. 아티팩트 디렉토리(`<appDataDir>/brain/<conversation-id>/task.md`)에 저장합니다.

## task.md 형식 예시

```markdown
# 프로젝트명 - 작업 목록

## 1. 단계 이름
- [ ] 세부 작업 A
- [ ] 세부 작업 B (`파일경로` — 설명)

## 2. 다음 단계 이름
- [ ] 세부 작업 C
- [ ] 세부 작업 D
```

## 진행 상황 업데이트 규칙

- 작업 시작 시: `- [ ]` → `- [/]` (진행 중)
- 작업 완료 시: `- [/]` → `- [x]` (완료)
- `task_boundary` 호출과 함께 `task.md`를 업데이트하여 동기화를 유지합니다.
